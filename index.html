Underwriter-Controlled Reprocess & COM File Linking

Nothing reprocesses without you.

A broker sends a revised vehicle schedule against an open deal. Today the AI engine quietly re-runs and overwrites the underwriter's work. This feature stops that: every file links to the Communication, a follow-up just rings a bell, and the underwriter chooses exactly which lines a reprocess may touch — with premium and hand-entered data protected. The engine logic is already built. These five concepts explore how underwriters drive it from the Submissions AI Engine Status card.

● Today

A follow-up email silently re-runs the engine and overwrites edited exposures, schedules and rates. No warning, no choice.

→
● With this feature

The email rings a bell. The underwriter squares up the files, then reprocesses only the lines they opt in. Everything else is untouched.

What's being displayed

The engine behavior, already built

Each decision below is implemented in the engine and Salesforce; the concepts only surface it. Traced to Nate's RFC (reprocess_control_and_com_file_linking_rfc) and the Jul-13 underwriter workshop.

D1

Files live on the Communication

Emailed and unzipped files link to the COM, not just the email — every document is in one place.

D2

No silent reprocess

A follow-up email no longer re-runs the AI. The underwriter gets a notification bell instead.

D3

You square up the files

Standard files list — remove superseded versions, upload extras directly. No email required.

D4

Protected from accidental delete

"Remove from record" unlinks; only admins can permanently delete an attached intake file.

D6

Choose what gets updated

An opt-in per line and per program design — all off by default. Salesforce writes only what you turn on.

D7

Premium is protected

Estimated / Proposed Bound Premium is never overwritten on reprocess — kept as an early estimate you refine.
